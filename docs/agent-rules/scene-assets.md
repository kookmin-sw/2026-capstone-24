# 씬 및 에셋 규칙

> 이 파일은 `.unity`, `.prefab`, `.asset` 파일을 수정하거나 Unity 씬·프리팹·에셋을 다루는 작업 시에만 읽으세요.

## 씬·에셋 규칙
- 작업이 명시적으로 샘플 구성을 대체하는 경우가 아니라면, 기존 Unity 샘플 wiring을 유지하는 쪽을 우선합니다.
- 샘플 에셋을 가볍게 이동하거나 이름을 바꾸지 않습니다. 많은 프리팹 및 씬 참조가 GUID에 민감합니다.
- 수정 범위는 좁게 유지하고 Unity 직렬화와 호환되도록 작업합니다. `.unity`, `.prefab`, `.asset` 파일에 불필요한 변동을 만들지 않습니다.
- 씬이나 프리팹을 수정할 때는 직렬화 데이터를 바꾸기 전에 대상 오브젝트 경로와 컴포넌트 할당 상태를 검증합니다.
- 씬 변경 전에 `Assets/Scenes/SampleScene.unity`에 아직 저장되지 않았거나 현재 작업과 무관한 사용자 수정이 이미 있는지 확인합니다.
- 프리팹 변경이나 스크립트 변경으로 해결 가능한 경우에는 넓은 범위의 직접적인 씬 재작성보다 그 방식을 우선합니다.
- 작업에 생성된 스크린샷이나 캡처가 필요하면 `Assets/Screenshots`처럼 `.gitignore`로 이미 제외된 경로 아래에 둡니다.
- `.meta` 파일은 보존하며, 명시적으로 필요한 경우가 아니라면 에셋을 삭제 후 재임포트해서 다시 생성하지 않습니다.
- Unity 도메인 리로드나 컴파일 이후에는 이전 씬 인스턴스 ID를 재사용하지 않습니다. 검증을 계속하기 전에 씬 오브젝트를 다시 조회합니다.
- 물리, 렌더러, 콜라이더 동작을 변경할 때는 먼저 프리팹과 씬의 인스펙터 설정을 확인하고, 런타임 오버라이드는 최후의 수단으로만 사용합니다.
