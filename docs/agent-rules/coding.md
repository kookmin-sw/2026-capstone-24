# 코딩 규약

> 이 파일은 `Assets/**/*.cs` 같은 C# 스크립트를 작성하거나 수정하는 작업 시에만 읽으세요.

- 새 코드는 `Assets/Hands/Scripts/`, `Assets/Instruments/_Core/Scripts/`처럼 **도메인 폴더 안의 `Scripts/` 서브폴더**에 런타임/에디터 로직을 C#으로 작성합니다.
- 기존 Unity 관례를 따릅니다: 인스펙터 연결을 위한 serialized field, 명시적인 컴포넌트 의존성, 최소한의 매직 스트링 사용.
- MonoBehaviour의 책임은 좁게 유지합니다. 복잡도가 커지면 씬에 묶인 스크립트에서 재사용 가능한 로직을 분리합니다.
- 코드에서 인스펙터 작성 가능 프로퍼티를 바꾸기 전에, 의도한 동작이 프리팹 또는 씬 직렬화 값에 있어야 하는 것은 아닌지 먼저 확인합니다.
- 특히 `Rigidbody.isKinematic`, `useGravity`, `interpolation`, `constraints`, `Collider.enabled`, `Renderer.enabled` 같은 컴포넌트 기본값은, 플레이 중 실제로 게임플레이가 그 값을 바꾸는 경우가 아니라면 런타임 소유 상태로 취급하지 않습니다.
- 어떤 컴포넌트 프로퍼티가 반드시 런타임 소유여야 한다면, 왜 인스펙터 데이터에 남길 수 없는지 드러나도록 명확한 메서드 이름이나 짧은 주석으로 그 소유권을 코드에서 분명히 합니다.
- 사용자가 더 풍부한 런타임 진단이나 디버깅 지원을 명시적으로 요청하지 않았다면, 로그 개선만을 위해 경고, 오류, 진단 상태 추적 로직을 추가하지 않습니다.
- Unity 동작이나 XR wiring이 자명하지 않은 곳에만 짧은 주석을 추가합니다.
- 아직 샘플 비중이 큰 프로젝트에서는 추측성 추상화를 피합니다.
