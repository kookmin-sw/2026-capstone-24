# 정합 작업 규칙

- 정합 작업은 센서, 콜라이더, 앵커, 시각 오브젝트를 실제 타깃 Transform/크기/방향에 맞추는 작업으로 본다.
- 편집 전 반드시 대상 범위를 잠근다. `prefab 자산`, `scene instance`, `둘 다` 중 무엇을 수정하는지 먼저 명시한다.
- 동일 도메인 오브젝트가 여러 개 있으면 이름만으로 추정하지 말고 전체 경로로 대상을 고정한다.
- 작업 시작 전 정합 대상과 소스 오브 트루스(예: 실제 키, end 마커, 본, 기준 메시)를 먼저 확인한다.
- 소스 오브 트루스는 1~2개 샘플에서 일반화하지 말고, 전체 대상 집합을 먼저 분류하고 집계한 뒤 그룹별 기준을 확정한다.
- 가능하면 수작업보다 규칙 기반 자동화를 우선한다. 반복될 가능성이 있으면 에디터 도구나 재현 가능한 절차를 만든다.
- 검증은 기본적으로 스크린샷과 수치 검증을 함께 사용한다.
- 스크린샷은 전체 뷰와 문제 구간 클로즈업을 함께 남긴다.
- 수치 검증은 대응 쌍 개수, 위치/회전/크기 오차, 누락 항목을 확인 가능한 형태로 남긴다.
- 완료 선언 전에는 수치 검증과 실제 사용자 시점의 시각 검증이 모두 끝나야 한다.
- 수치와 화면이 충돌하면 완료로 판단하지 말고, 기준이 잘못됐는지, 대상이 잘못됐는지, `scene instance`와 `prefab asset`이 불일치하는지 다시 점검한다.
- 프리뷰용 임시 오브젝트를 만들었다면 작업 종료 전에 제거하고, 씬이나 프리팹에 불필요한 흔적을 남기지 않는다.
