# Note Visual Fidelity

**Parent:** [`_index.md`](../_index.md)

## Why

현재 모든 노트가 동일한 높이의 짧은 막대로 표시된다. 차트에서 4분음표와 2분음표가 같은 크기로 보이면 플레이어가 음표 길이를 시각적으로 파악할 수 없어 언제 손을 떼야 하는지 알기 어렵다. 또한 단순 2D 사각형 방식은 디자인 커스터마이징이 불가능해 향후 다양한 스킨을 제공하기 어렵다.

## What

- **노트 길이 시각화**: 노트의 세로 높이가 차트에 기록된 음표 길이(durationTicks)에 비례한다. 4분음표는 8분음표보다 2배 높고, 2분음표는 4분음표보다 2배 높다. 노트가 낙하하는 속도는 변하지 않으며, 긴 노트는 더 오랫동안 판정선에 걸쳐 있게 된다.
- **3D 프리팹 스킨**: 노트 시각화 단위를 교체 가능한 3D 프리팹으로 구성한다. 기본 스킨이 제공되며, 스킨 에셋을 바꾸면 이후 스폰되는 노트에 즉시 반영된다. 흰 건반 노트와 반음 노트는 각각 다른 스킨 슬롯을 가질 수 있다.
- **노트 높이 클리핑**: 노트 높이가 패널 높이를 초과하면 패널 상단에서 클리핑되어 표시된다 (스크롤 없음).

## Behavior

- **Given** 차트에 C4 4분음표 → D4 4분음표 → E4 2분음표가 정의됨
  **When** 세션이 실행됨
  **Then** C 노트와 D 노트는 같은 높이로 낙하하고, E 노트는 그 2배 높이로 낙하한다. 플레이어 시점에서 E 노트가 판정선을 지나는 시간이 D 노트의 2배다.

- **Given** 차트에 C4 4분음표 다음 D4 4분음표가 연속으로 정의됨
  **When** 세션이 실행됨
  **Then** C 노트의 아래 끝이 판정선에 도달하는 시각과 D 노트의 아래 끝이 상단에서 나타나는 시각이 일치하여, 두 노트 사이에 시각적 공백이 생긴다.

- **Given** 흰 건반 스킨 슬롯에 스킨 에셋 B가 설정됨
  **When** 세션이 시작됨
  **Then** 흰 건반 노트는 모두 B 스킨으로 표시된다.

- **Given** 세션이 진행 중
  **When** 스킨 에셋이 교체됨
  **Then** 이미 낙하 중인 노트는 기존 스킨을 유지하고, 이후 스폰되는 노트부터 새 스킨이 적용된다.

- **Given** 차트에 4분음표의 4배 이상인 긴 음표가 정의됨
  **When** 세션이 실행됨
  **Then** 노트 높이가 패널 높이를 초과하면 패널 상단에서 클리핑되어 표시된다.

## Out of Scope

- 스킨 선택 UI (메뉴/설정 화면).
- 노트 가로 너비의 스킨 제어 — 너비는 레인 구성(흰 건반 80%, 반음 55%)에 따른다.
- 홀드 노트(누르고 있어야 하는 노트)의 별도 판정 로직 — 이 spec은 시각 표현만 다룬다.
- 애니메이션·파티클 효과.
