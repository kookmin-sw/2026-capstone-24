# Drum Note Display

**Parent:** [`_index.md`](../_index.md)

## Why

피아노는 모든 건반을 단일 평면 패널에 노트 라인으로 표시하지만, 드럼은 악기 특성 상 각 파츠(킥, 스네어, 하이햇 등)가 공간적으로 분리된 위치에 있다. 단일 평면 패널로 드럼을 표현하면 어떤 파츠를 쳐야 하는지 직관적으로 전달되지 않는다. 각 드럼 파츠의 실제 VR 위치에 노트 라인 패널을 배치하면, 플레이어는 시선을 해당 파츠 방향으로 자연스럽게 유도받아 리듬게임을 플레이할 수 있다.

## What

드럼 악기를 잡은 상태에서 리듬게임 세션이 시작되면, 고정 드럼 파츠 세트의 각 파츠 위치에 독립적인 노트 라인 패널이 표시된다. 각 패널은 해당 파츠를 향해 노트가 낙하하며, 판정선은 드럼 파츠 바로 위에 위치한다.

- 지원하는 드럼 파츠 세트는 고정이다: 킥(Kick), 스네어(Snare), 하이햇(Hi-Hat), 크래시(Crash), 탐(Tom) 등 표준 드럼 세트 구성.
- 각 파츠 패널은 플레이어가 서 있는 위치 기준, 해당 파츠의 실제 VR 공간 위치에 배치된다.
- 차트에서 드럼 트랙의 MIDI 노트 번호는 GM 드럼 맵에 따라 파츠별로 분류된다 (아래 표).
- 플레이어가 잡은 악기가 드럼일 때만 이 디스플레이가 활성화된다. 피아노의 평면 패널과는 별개 시스템이다.

| 파츠 | MIDI 노트 번호 |
|---|---|
| Kick | 36 |
| Snare | 38 |
| Hi-Hat | 42 (Closed), 46 (Open) |
| Crash | 49 |
| Tom | 45, 47, 48, 50 |

## Behavior

- **Given** 플레이어가 드럼을 잡고 세션을 시작할 때
  **When** 세션 리드인이 시작될 때
  **Then** 각 드럼 파츠의 실제 위치에 노트 라인 패널이 나타나고, 해당 파츠에 해당하는 노트가 낙하하기 시작한다

- **Given** 드럼 세션이 진행 중일 때
  **When** 차트에서 킥 노트가 스폰될 때
  **Then** 킥 패널에만 노트가 낙하하며, 다른 파츠 패널에는 영향 없음

- **Given** 드럼 세션이 진행 중일 때
  **When** 플레이어가 스네어를 치는 타이밍에 스네어 패널에 노트가 판정선에 도달할 때
  **Then** Perfect / Good / Miss 판정이 스네어 파츠 기준으로 발생한다

- **Given** 차트에 드럼 트랙이 있고 플레이어가 드럼을 잡지 않은 경우
  **When** 세션이 시작될 때
  **Then** 드럼 트랙은 자동 반주로 발화되고, 드럼 노트 패널은 표시되지 않는다

## Out of Scope

- 피아노 등 다른 악기의 노트 디스플레이 변경
- 드럼 파츠 세트를 동적으로 감지하거나 차트 채널 정보로 결정하는 로직 (고정 세트 사용)
- 드럼 파츠 패널의 배경/프레임 커스터마이징
- 판정(Perfect / Good / Miss) 타이밍 로직 변경 (Judgment sub-spec 담당)
- 드럼 파츠의 VR 위치를 씬에서 동적으로 계산하는 방식 결정 (plan에서 구체화)
- 드럼 파츠 패널의 법선 방향(플레이어를 향하는 자동 회전 vs 고정 방향) 결정 (plan에서 구체화)
