# Chart Import

**Parent:** [`_index.md`](../_index.md)

## Why

차트는 Unity 외부에서 편집되는 텍스트 파일이지만, 런타임은 빠른 접근과 참조 관리를 위해 Unity 자산 형태로 다뤄야 한다. 또한 차트 파일이 추가/수정되었을 때 자동으로 갱신되어야 작성자가 매번 수동 변환을 반복하지 않는다. 포맷 오류가 있을 때는 작성자가 어디가 잘못됐는지 인지할 수 있어야 한다.

## What

프로젝트에 차트 파일이 존재하면, Unity 에디터는 그 파일을 런타임이 사용 가능한 자산 형태로 자동 변환한다. 파일이 추가/수정/삭제되면 변환 결과도 그에 맞춰 갱신된다. 포맷 오류는 사용자가 인지할 수 있는 형태로 보고되며, 잘못된 차트는 런타임이 잘못 사용하지 않도록 차단된다.

## Behavior

- **Given** 새 차트 파일이 프로젝트 폴더에 추가됨
  **When** Unity가 자산을 갱신함
  **Then** 그 차트가 런타임에서 참조 가능한 자산으로 등장한다.

- **Given** 기존 차트 파일이 텍스트 에디터에서 수정됨
  **When** Unity가 변경을 감지함
  **Then** 자산이 새 내용으로 갱신된다.

- **Given** 차트 파일에 포맷 오류가 있음
  **When** 임포트가 시도됨
  **Then** 작성자가 인지할 수 있는 방식으로 오류가 표시되며, 잘못된 자산이 런타임에서 사용되지 않는다.

- **Given** 차트 파일이 프로젝트에서 삭제됨
  **When** Unity가 자산을 갱신함
  **Then** 대응하는 자산도 사라진다.

## Out of Scope

- 차트 텍스트 포맷 자체의 사양 → `chart-format`.
- 런타임에서 자산이 어떻게 재생/판정되는지 → `timing-clock` / `judgment` / `accompaniment`.
- 차트 외부 작성/변환 도구.
