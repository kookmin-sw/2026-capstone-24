# Accompaniment

**Parent:** [`_index.md`](../_index.md)

## Why

곡에는 여러 악기 트랙이 있고 플레이어는 한 악기만 잡는다. 곡에 해당하는 별도 오디오 파일이 없기 때문에, 플레이어가 잡지 않은 트랙을 게임이 시간에 맞춰 직접 발화하지 않으면 그 트랙의 소리는 아예 들리지 않는다. 즉, 자동 반주가 없으면 곡으로 들리지 않는다.

## What

세션이 진행되는 동안, 플레이어가 잡지 않은 트랙(자동 반주 트랙)의 각 노트는 마스터 시계 기준 정해진 시각에 해당 악기로 발화된다. 발화는 기존 사운드 파이프라인을 통해 일어나, 결과적으로 그 악기에서 직접 연주된 것과 동일한 사운드가 난다. 차트에 정의된 트랙이지만 씬에 대응하는 악기 인스턴스가 없다면 그 트랙은 발화되지 않는다.

## Behavior

- **Given** 자동 반주 트랙에 시각 t에 예정된 노트가 있고, 그 트랙의 채널에 매핑된 악기가 씬에 존재함
  **When** 마스터 시계가 t에 도달
  **Then** 그 악기가 해당 노트(MIDI 노트 번호, 세기, 길이)를 즉시 발화한다.

- **Given** 자동 반주 트랙의 채널에 매핑된 악기가 씬에 없음
  **When** 그 트랙의 노트 시각이 도래
  **Then** 발화가 일어나지 않으며, 다른 트랙은 영향을 받지 않는다.

- **Given** 여러 자동 반주 트랙이 같은 시각에 노트를 가짐
  **When** 그 시각이 도래
  **Then** 각 악기에서 독립적으로 동시에 발화된다.

- **Given** 세션이 일시정지/종료됨
  **When** 자동 반주 트랙의 다음 예정 시각이 더 이상 도래하지 않음
  **Then** 자동 발화가 멈춘다.

## Out of Scope

- 플레이어 입력 채점 → `judgment`.
- 마스터 시계 → `timing-clock`.
- 차트의 채널-악기 매핑 정의 → `chart-format`.
- 사운드 합성/재생 메커니즘 자체 (기존 악기 시스템에 위임).
- 트랙별 음량/믹스 조절 (추후 리듬게임 시작 전 설정 UI에서 처리 예정).
- 트랙별 소리 on/off 선택 UI (추후 리듬게임 시작 전 설정 UI에서 처리 예정).
