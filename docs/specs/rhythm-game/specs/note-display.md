# Note Display

**Parent:** [`_index.md`](../_index.md)

## Why

리듬게임 세션이 진행되는 동안 플레이어는 자신이 연주할 노트의 타이밍을 시각적으로 파악해야 한다. 악기를 바라보는 자연스러운 시선 방향에 노트 정보가 함께 표시되지 않으면, 플레이어는 HMD 화면과 악기 사이에서 시선을 계속 전환해야 하므로 몰입과 조작 모두 방해된다. 악기 표면에 직접 붙은 UI가 이 문제를 해결한다.

## What

리듬게임 세션이 활성화된 동안, 각 악기에 레인과 낙하 노트로 구성된 노트 디스플레이 패널이 표시된다. 패널은 악기 표면(또는 파츠 표면)에 월드 스페이스 캔버스 형태로 부착된다. 노트가 판정선에 도달하는 순간 플레이어의 연주를 기다리며, 판정 결과(Perfect / Good / Miss)는 패널 안에 인라인으로 표시된다.

- 모든 악기에 동일한 레인+낙하 방식을 사용한다.
- 악기 형태에 따라 패널 배치가 달라진다: 피아노처럼 건반이 이어진 악기는 하나의 연속 패널로, 드럼처럼 파츠가 분리된 악기는 각 파츠 표면에 개별 패널을 배치하며, 패널의 위치·방향은 파츠 형태에서 런타임에 자동 계산된다.
- 레인 수와 레인-MIDI 노트 대응은 악기별로 사전 정의된 악기 메타데이터에 따른다. 차트는 레인 번호로만 참조하며, 대응 방식 자체는 차트에 담지 않는다.
- 채점 대상 트랙에 화음(동시 다발 노트)이 있으면, 해당 노트 각각이 대응 레인에 동시에 표시된다.
- 세션이 비활성 상태이면 패널은 표시되지 않는다.

## Behavior

- **Given** 리듬게임 세션이 시작됨
  **When** 플레이어가 채점 대상 악기를 바라봄
  **Then** 악기 표면에 노트 디스플레이 패널이 나타나고 노트가 레인을 따라 내려오기 시작한다.

- **Given** 노트 디스플레이가 표시 중
  **When** 노트가 판정선에 도달하는 타이밍에 플레이어가 해당 입력을 수행함
  **Then** Perfect / Good 중 하나로 판정되고, 결과 텍스트가 패널 안에 팝업된 뒤 짧게 페이드아웃된다. 지속 시간·애니메이션은 plan에서 결정한다.

- **Given** 노트 디스플레이가 표시 중
  **When** 노트가 판정선을 지나도록 입력이 없었음
  **Then** Miss로 판정되고, 결과 텍스트가 패널 안에 팝업된 뒤 짧게 페이드아웃된다.

- **Given** 리듬게임 세션이 종료됨
  **When** 세션 종료 신호가 전달됨
  **Then** 노트 디스플레이 패널이 사라진다.

- **Given** 드럼처럼 파츠가 분리된 악기
  **When** 세션이 시작됨
  **Then** 각 파츠(예: 스네어, 하이햇) 표면에 개별 패널이 배치되고, 파츠별 레인이 독립적으로 표시된다.

## Out of Scope

- 세션 시작 / 일시정지 / 종료 조작 → `session-flow`.
- 곡 선택 메뉴 UI.
- 결과 화면(점수 합산 / 등급 표시).
- 자동 반주 트랙의 시각화 (반주 트랙 노트는 표시 안 함).
- 판정 로직 자체 → `judgment`.
