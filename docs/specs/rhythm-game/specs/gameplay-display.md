# Gameplay Display

**Parent:** [`_index.md`](../_index.md)

## Why

리듬게임의 핵심 코드(판정·세션·노트 낙하)가 구현되어 있지만, VR 씬 안에서 플레이어가 실제로 경험하는 연결이 완성되지 않았다. 피아노 앞 적절한 위치에 UI가 배치되고, 피아노를 치면 즉각적인 시각 피드백이 돌아와야 비로소 리듬게임이 플레이 가능한 형태로 완성된다.

## What

- 노트 낙하 디스플레이 캔버스가 **악기 GameObject의 자식**으로 배치된다. 피아노·드럼 등 모든 악기에 동일하게 적용되며, 악기를 이동하면 UI가 함께 따라간다.
- 세션이 시작되면 VR 플레이어 시점에서 노트가 위에서 아래로 떨어지는 것이 보인다.
- 플레이어가 악기를 연주하면 판정 결과(Perfect / Good / Miss)가 디스플레이에 즉시 표시된다.

## Behavior

- **Given** 리듬게임 세션이 시작됨
  **When** 첫 노트 스폰 시각이 됨
  **Then** 피아노 앞 월드 공간에 배치된 디스플레이에서 흰 사각형 노트가 위에서 판정선으로 낙하하기 시작한다.

- **Given** 노트가 판정선 부근에 있음
  **When** 플레이어가 해당 피아노 건반을 침
  **Then** Perfect / Good / Miss 중 하나의 텍스트가 디스플레이에 짧게 표시된 후 사라진다.

- **Given** 플레이어가 VR 헤드셋을 착용하고 피아노 앞에 섬
  **When** 세션이 진행 중임
  **Then** 디스플레이가 피아노 정면 시야 안에 위치해 고개를 크게 돌리지 않아도 노트와 판정 팝업이 보인다.

- **Given** 세션이 종료됨
  **When** StopSession이 호출됨
  **Then** 디스플레이가 비활성화되어 피아노 앞에서 사라진다.

## Out of Scope

- 점수 합산·결과 화면 (별도 sub-spec)
- 세션 진입 메뉴 UI (Session Flow sub-spec 담당)
- 드럼 등 다른 악기의 멀티패널 배치 (DrumNoteDisplayAdapter 로직은 이미 구현됨, 씬 배치는 추후)
- 캔버스 외형 디자인·애니메이션 개선
