# Stick No-Penetration

**Parent:** [`_index.md`](../_index.md)

## Why

스틱이 단순히 손의 위치를 그대로 따라가기만 하면 드럼 표면·심벌 같은 타격면을 그대로 통과해버린다. 통과는 두 가지 비용을 만든다.

1. "어디서부터 표면이고 어디부터 친 것인가"의 시각·물리 단서가 사라져 타격 정확도와 박자감이 떨어진다.
2. VR 공간에서 드럼킷의 실재감이 깨진다.

또한 스틱은 손과 달리 길고 가는 객체라 통과가 시각적으로 더 두드러지며, 드럼은 본질적으로 "표면을 친다"는 행위가 핵심이라 통과 방지 없이는 연주 자체가 성립하지 않는다.

## What

- 사용자가 잡고 있는 스틱이 드럼킷 부품(드럼 표면, 심벌, 림 등) 안쪽으로 들어가지 않는다.
- 스틱이 표면에 닿은 상태에서도 사용자의 입력은 끊기지 않으며, 스틱은 표면 위에서 멈춰 보인다.
- 표면을 떠나면 스틱은 다시 사용자의 손 위치를 자연스럽게 따라간다.
- 통과 방지의 결과로 발생하는 사용자의 입력 위치와 스틱 시각 위치 간의 어긋남은 허용된다.

## Behavior

- **Given** 사용자가 양손에 스틱을 들고 있고 스틱이 드럼킷 표면 위 공간에 있다.
  **When** 사용자가 손을 드럼킷 표면 안쪽으로 더 밀어 넣는다.
  **Then** 스틱은 표면에서 멈춰 보이며, 표면 안쪽으로 들어가지 않는다.

- **Given** 스틱이 드럼킷 표면에 닿아 멈춰 있다.
  **When** 사용자가 손을 표면에서 떼어 위로 들어 올린다.
  **Then** 스틱은 다시 손의 위치를 따라가 자연스럽게 표면에서 떨어진다.

- **Given** 사용자가 한 손의 스틱으로 드럼킷 표면을 빠르게 연속 타격한다.
  **When** 다음 타격 입력이 들어온다.
  **Then** 연주에 지장을 줄 만한 레이턴시·떨림 없이 다음 타격이 인식된다.

- **Given** 사용자의 양손 스틱이 같은 드럼/심벌 영역에서 만난다.
  **When** 두 스틱의 위치가 서로 겹친다.
  **Then** 스틱끼리는 서로 통과하며 막히지 않는다 (스틱-스틱 충돌은 통과 방지 대상이 아니다).

## Trade-offs

본 sub-spec의 의도된 적용 범위는 **드럼킷 부품에 한해 통과 방지**다. 그러나 plan 단계에서 다음 단순화 안이 검토될 수 있다.

- **광범위 콜라이더 통과 금지**: 스틱이 드럼킷뿐 아니라 환경 콜라이더(바닥/벽/다른 객체)와도 통과하지 않게 처리. 카테고리 필터 없이 전 충돌을 허용해 구현·셋업 비용이 낮다. 단점: 사용자가 anchor 자세에서 손을 크게 흔들 때 의도하지 않은 환경 객체와 부딪힐 수 있다.
- **드럼킷 부품 한정**: 사용자 의도와 일치하지만 콜라이더 카테고리 분리·레이어 셋업 비용이 든다.

어느 안을 채택할지는 plan 단계에서 비용과 사용자 관찰 영향을 비교해 결정한다. plan 채택 결과가 본 sub-spec의 What과 어긋나면(예: 환경까지 막혔을 때 사용자가 어색함을 강하게 관찰한다면) 본 sub-spec을 갱신한다.

## Out of Scope

- 스틱이 드럼에 닿았을 때의 노트 트리거(타격 깊이/속도 기반 인식) — 본 sub-spec은 통과 방지·멈춤만 책임진다.
- 손 자체와 드럼 표면의 통과 방지 → [`hands/specs/02-instrument-no-penetration.md`](../../hands/specs/02-instrument-no-penetration.md).
- 햅틱 피드백 정책.
- 스틱 자체의 부서짐·파손 같은 물리 시뮬레이션.
- detach된 스틱이 외부 공간에서 환경과 충돌하는 거동 — detach 직후 즉시 사라지므로 발생하지 않음.
