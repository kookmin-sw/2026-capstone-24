# 유저 데이터 영속화

**Parent:** [`_index.md`](../_index.md)

## Why

인증만으로는 유저 정보가 세션이 끊어지면 사라진다. 유저 프로필, 접속 이력 등을 DB에 저장해야 재접속 시에도 동일한 유저로 식별하고 일관된 경험을 제공할 수 있다.

## What

서버에서 인증된 유저 정보를 DB에 저장하고 조회하는 기능을 제공한다. 최초 로그인 시 유저 레코드를 생성하고, 재접속 시 기존 레코드를 조회한다.

## Behavior

- **Given** 최초 로그인한 유저가
  **When** 인증에 성공하면
  **Then** 유저 레코드가 DB에 생성된다.

- **Given** 이미 가입된 유저가
  **When** 재로그인하면
  **Then** 기존 레코드가 조회되어 동일한 유저로 식별된다.

- **Given** 서버가 재시작되어도
  **When** 유저가 재접속하면
  **Then** 이전 유저 정보가 그대로 유지된다.

## Out of Scope

- 유저 프로필 수정 UI
- 유저 삭제·탈퇴 기능
- 게임 플레이 통계·기록 저장
