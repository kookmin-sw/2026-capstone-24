# Left Pinch Gesture

**Parent:** [`_index.md`](../_index.md)

## Why

악기를 잡지 않은 상태의 사용자가 UI를 빠르게 호출할 트리거가 필요하다. 컨트롤러 버튼에 의존하면 핸드 트래킹이나 노컨트롤러 환경에서 일관성이 깨지고, 양손 어디서든 같은 모양의 핀치를 트리거로 쓰면 악기 잡기·놓기 같은 다른 손 동작과 충돌할 수 있다. 왼손에 한정한 메타 핀치 제스쳐는 이런 충돌을 줄이면서 모든 입력 환경에서 동일하게 발화될 수 있는 호출 채널을 제공한다.

## What

왼손의 엄지와 검지가 사용자를 마주보는 방향으로 꼬집히는 동작을 별도의 입력 이벤트로 발화한다. "사용자를 마주보는 방향"은 왼손이 사용자의 시야 안에 있고 동시에 왼손 손바닥 법선이 사용자 쪽을 향하는 두 조건을 모두 만족하는 상태로 정의한다. 핀치 형성·해제는 시작·해제 임계를 분리해 채터링을 방지한다(시작보다 해제가 더 멀리 떨어져야 함). 이 sub-spec은 hands 피처 안에서 그 이벤트를 정의·검출하는 책임만 가진다. 이 이벤트를 받아 무엇을 표시하거나 실행할지는 본 sub-spec의 책임이 아니며, 다른 피처(예: session-panel)가 소비한다. 컨트롤러 입력 환경에서는 동등 매핑된 컨트롤러 입력으로 같은 이벤트가 발화되며(기본 매핑: 좌측 컨트롤러 메뉴 버튼 채널, 보조: 좌측 보조 버튼 채널), 소비측은 입력 채널을 구분하지 않는다.

## Behavior

- **Given** 사용자가 왼손을 자기 시야 안에 두고 있고 왼손에 어떤 악기도 잡혀 있지 않음
  **When** 왼손 엄지와 검지 끝이 서로 닿아 핀치 모양이 형성됨(꼬집는 면이 사용자 쪽을 향함)
  **Then** 핀치 시작 이벤트가 한 번 발화된다.

- **Given** 핀치가 유지되고 있음
  **When** 두 손가락 끝이 다시 임계 거리 이상 벌어짐
  **Then** 핀치 해제 이벤트가 한 번 발화된다.

- **Given** 사용자가 왼손에 어떤 악기를 잡고 있음
  **When** 동일한 손가락 모양이 발생
  **Then** 핀치 이벤트는 발화되지 않는다.

- **Given** 사용자가 왼손을 자기 시야 밖에 두고 있음(또는 꼬집는 면이 사용자 반대 방향)
  **When** 동일한 손가락 모양이 발생
  **Then** 핀치 이벤트는 발화되지 않는다.

## Out of Scope

- 핀치 이벤트를 받아 어떤 UI를 띄우거나 동작을 수행할지 — 소비측 피처(예: [`session-panel/specs/01-anchoring.md`](../../session-panel/specs/01-anchoring.md))의 책임.
- 오른손 핀치, 양손 핀치, 그 외 메타 제스쳐.
- 핀치를 통한 일반 UI 포인팅·선택(이번 sub-spec은 호출 트리거 한 종류만 다룸).
- 핸드 트래킹·컨트롤러 입력 채널의 추상화 자체.
- 짧은 시간 내 반복 핀치(더블 핀치)에 별도 의미를 부여하는 동작 — 본 sub-spec은 single 핀치 시작/해제 이벤트만 다룬다.
